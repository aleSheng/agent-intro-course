## 模块一：从"会聊天"到"会干活"——你可能低估了现在的AI (12分钟，9页)

> 大部分人对AI的认知还停留在"问它问题，它给我答案"。但2025-2026年发生了一件本质变化：AI从只有"嘴巴"长出了"手"。

### 关键认知升级
*   **对话式AI（ChatBot）**：你问一句，它答一句。它活在一个文本框里。
*   **智能体（Agent）**：你给一个目标，它自己规划步骤、调用工具、读写文件、执行终端命令、上网搜索、遇到报错自己修——直到任务完成。它活在你的**整台电脑**里。
*   一句话区别：ChatBot 是军师，Agent 是将军带兵打仗。

### Background Agent：下一步进化
> 来源：background-agents.com

Agent 的进化方向：从"你盯着它干"到"它在后台自主工作"。

| 阶段 | 模式 | 你的角色 |
|------|------|---------|
| **ChatBot** | 你问一句，它答一句 | in the seat（你坐在驾驶座） |
| **Agent** | 你给目标，它当面干完 | in the loop（你在循环内监督） |
| **Background Agent** | 事件触发，后台自主工作，完成后提交审核 | on the loop（你在循环外审批） |

### Agent Skill：给 AI 装上"技能包"

Agent Skill 是一个包含指令、脚本和资源的文件夹，Agent 可以**动态发现、按需加载**，从而在特定任务上表现得更好。

**MCP vs Skill——两个互补的东西：**

|  | MCP | Agent Skill |
|--|-----|------|
| 功能 | 给模型**提供数据** | 教模型**如何处理任务** |
| 举例 | 查询内部文档、读取数据库 | 代码审查清单、结构化写作流程 |
| 本质 | 程序（需要运行服务） | 文件（放在文件系统即可） |
| 类比 | 给厨师送食材 | 给厨师一本菜谱 |

**渐进式披露（Progressive Disclosure）：** Skill 的核心架构是三层按需加载：
1. **元数据层**（~100 tokens/skill）：名称+描述，启动时始终加载
2. **指令层**（<5000 tokens）：SKILL.md 正文，技能被激活时加载
3. **资源层**（不限）：scripts/references/assets，按需读取或执行

效果：Agent 可以"知道"数百个技能的存在，但只在需要时加载具体内容，上下文窗口零浪费。

**生态爆发：** Anthropic 2025.12 将 Agent Skills 发布为开放标准，数周内 20+ 平台（Codex CLI、Gemini CLI、Cursor、Copilot 等）采用同一格式。2026.03 SkillsMP 市场累计 27 万+ 技能。

## 模块二：AI Agent 实战 (35分钟，13页)

> 这一部分是全课的核心。通过三个板块，展示Agent能力的边界在哪里、走在最前面的人在怎么用它们。

### Section A: 终端智能体（4页）

#### OpenClaw 与终端智能体

*   **故事线**：OpenClaw 的作者并非传统程序员出身。他从第一行代码到产品上线，全程通过 AI Agent 以自然语言"指挥"完成。技术原理是 LLM + 截图 + 鼠标脚本的循环。**很火、很有争议，配置也很复杂。**

#### 更多选择

OpenClaw 复杂度高，但终端智能体生态正在快速扩展：

| 工具 | 特色 |
|------|------|
| **nanoClaw** | 极简入门，几行配置就能跑 |
| **zeroclaw** | 零配置，开箱即用 |
| **Kimi Claw** | 月之暗面出品，超长上下文 |
| **MaxClaw** | 极致自动化，端到端完成任务 |
| **Manus** | 云端全能助手，无需本地环境 |

#### Agentic Loop

Agent 最核心的能力模式：写-测-错-改循环。Agent不怕出错，它会自己修。

#### 冷静看 OpenClaw

*   **批判性视角**：
    *   争议：截图操作效率低、易被UI变化干扰、占用大量 Token
    *   价值：降低了编程门槛、推动了开源 Agent 生态
    *   未来方向：更好的视觉编码器、Accessibility Tree 理解
*   **核心启发**：用批判性思维看待技术热点。

### Section B: Vibe Coding（4页）

#### Vibe Coding

*   **核心概念**：Vibe Coding = 跟着感觉写代码。你不需要学编程，只需要会描述需求。
*   **现场演示**：老师打开一个空目录，在终端里用纯自然语言，让 Agent 从零构建一个完整的小产品。
*   **重点展示 Agent 的工作过程**（而非只看最终结果）：
    *   Agent 如何自己规划文件结构
    *   写到一半自己运行、发现报错、阅读错误日志
    *   自动生成修复补丁、重新测试
    *   最终成功弹出可用产品

#### 人人都能 Vibe Coding

*   **非程序员的三步流程**：
    1.  **用自然语言描述需求** —— 告诉 AI 你想要什么，不需要写一行代码
    2.  **Agent 自动生成** —— AI 写代码、创建文件、安装依赖、运行调试
    3.  **预览并调整** —— 看到效果后继续用自然语言修改

#### Vibe Coding 启发

*   未来的"编程"更像是当产品经理——你负责提需求、验收质量、决定方向，Agent负责执行。
*   **一个人 + 一个下午 + 这些工具 = 一个真实可用的产品。**

### Section C: Vibe Research（5页）

#### Vibe Research

*   用 AI 重构你的**学习、研究与论文复现**方式。涵盖从知识获取到论文实验复现的全流程。

#### NotebookLM 演示

*   将几篇晦涩的学术论文扔进 NotebookLM，一键生成双人播客对谈音频——枯燥的内容变成了轻松的闲聊。

#### Kimi 超长上下文

*   把一整本教材/一个完整的GitHub仓库扔进去，直接向它提问，让它成为这份材料的"活字典"。

#### 复现论文四步法

1.  **喂论文给 AI** —— 把 PDF 扔进 Kimi / Claude，提炼核心算法、公式、数据流
2.  **生成实现计划** —— 让 AI 根据论文方法论，拆解出代码实现的步骤和文件结构
3.  **Agent 逐步编码** —— AI 自动写代码、跑实验、看报错、修 bug（Agentic Loop）
4.  **对比验证结果** —— 将实验输出与论文中的表格/图表对比，用 AI 分析差异原因并迭代

#### Vibe Research 启发

*   信息获取的方式正从"我去读"变成"它来讲给我听 / 我来直接问它"。
*   以前复现论文需要读懂每一行代码，现在只需要读懂论文本身。**核心能力回到了它该在的地方：理解问题本身，而不是工程实现。**

## 模块四：清醒认知——Agent 的能力边界与使用原则 (15分钟，8页)

> 不灌鸡汤，讲清楚三件事。

### 1. Agent 会"翻车"

*   **幻觉问题**：Agent 会一本正经地编造不存在的API、引用不存在的论文。它不知道自己不知道。
*   **权限风险**：终端 Agent 拥有读写文件、执行命令的权限。一个模糊的"清理不需要的文件"指令，可能导致它删掉你的毕业论文。
*   **死循环**：Agent 可能反复用同一个错误方法尝试修复同一个bug，陷入无限重试。

### 2. 安全底线

*   **沙盒隔离**：重要操作永远在容器/虚拟环境里运行，不要在宿主机上裸跑不受信任的Agent。
*   **Human-in-the-Loop**：涉及删除、支付、发布等不可逆操作，必须人工确认。像 Claude Code 默认就会在关键操作前暂停等你按 Y/N。
*   **不给敏感信息**：不要把密钥、密码直接写进 Prompt。

### 3. Agent 不只写代码

> 2026年2月：Claude + Palantir 在军事行动中完成了完整的 Agent Loop —— 11分23秒。

*   **同一套模式**：感知→分析→决策→执行→评估 —— 这和 Vibe Coding 里的 Agentic Loop 是完全相同的模式，只是作用域从代码变成了现实世界。
*   **完全不同的后果**：写代码出 bug 可以 revert。但当 Agent 的"执行权限"从文件系统扩展到**物理世界**，每一个决策都不可逆。
*   **核心警示**：技术无善恶，但**使用者的判断力**决定了它是工具还是武器。

### 4. 真正值钱的能力

| 你以为值钱的 | 真正值钱的 |
|-------------|-----------|
| 会写代码 | 能把模糊需求拆解成清晰的、可执行的子任务 |
| 熟悉某个框架的API | 能判断AI生成的100个方案里哪个最优、为什么 |
| 打字快 | 能用精准的自然语言描述你到底要什么（Context Engineering） |
| 什么都自己干 | 知道什么该交给AI、什么必须自己把关 |

### 5. Context Engineering

用精准自然语言描述需求——这才是AI时代的核心技能。不是"让 AI 听懂你"——而是"给 AI 足够的上下文，让它做出正确判断"。

## 结尾 (5分钟，4页)

*   **一个判断**：我们正处在这样一个时刻——"造东西"的门槛在急剧降低，但"造出好东西"的门槛从未降低。工具越强大，人的**品味（Taste）、判断力和问题定义能力**就越稀缺。
*   **一个建议**：今天回去，挑一个你生活中真实的低效场景（一个重复性的任务、一个你一直想做的小工具、一个你专业领域里的痛点），打开今天提到的任意一个工具，试着让AI帮你解决它。第一次可能不完美，但你会发现——**你已经有能力造东西了**。
*   **一句话**："AI won't replace you. But someone using AI will."
