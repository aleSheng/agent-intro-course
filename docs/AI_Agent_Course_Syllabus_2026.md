# 当"不会写代码的人"开始造软件——2026 AI智能体前沿实录

**课程时长**：90-120 分钟
**目标受众**：全校本科生（通识课，各专业背景，已有AI对话/写作/搜索经验）
**课程定位**：你们已经会用AI聊天和写作了。这堂课讲的是你们**还不知道的事**——AI Agent 正在如何重塑"造东西"这件事本身，以及走在最前面的人在怎么玩。

---

## 模块一：从"会聊天"到"会干活"——你可能低估了现在的AI (10分钟)

> 大部分人对AI的认知还停留在"问它问题，它给我答案"。但2025-2026年发生了一件本质变化：AI从只有"嘴巴"长出了"手"。

### 关键认知升级
*   **对话式AI（ChatBot）**：你问一句，它答一句。它活在一个文本框里。
*   **智能体（Agent）**：你给一个目标，它自己规划步骤、调用工具、读写文件、执行终端命令、上网搜索、遇到报错自己修——直到任务完成。它活在你的**整台电脑**里。
*   一句话区别：ChatBot 是军师，Agent 是将军带兵打仗。

### 2026 基座模型速览（1分钟过完）
> 在座各位日常用的模型可能就一两个。但前沿开发者会根据任务"换脑子"：

| 模型 | 核心优势 | 什么时候用它 |
|------|---------|-------------|
| **Claude Opus 4.6** | 代码精度 + 超长文本零遗漏 | 复杂工程项目、几十万字文档分析 |
| **Gemini 3.1 Pro** | 原生多模态（视频/音频/代码/文本） | 需要同时处理多种媒体格式的任务 |
| **GPT 5.3** | 综合推理天花板 | 跨学科复杂问题、模糊需求的创意发散 |
| **Qwen 3.5** | 开源 + 本土化 + 工具调用生态 | 企业私有部署、国内业务深度集成 |
| **Kimi 2.5** | 超长上下文记忆 | 一次性喂入整本书/整个代码库做分析 |

选模型就像选员工：不是选"最聪明的"，而是选"最适合这个岗位的"。

## 模块二：前沿案例拆解——那些"不可能"正在变成日常 (35分钟)

> 这一部分是全课的核心。通过4个真实的前沿案例，展示Agent能力的边界在哪里、走在最前面的人在怎么用它们。

### 案例1：OpenClaw——"不会写代码的人"如何造出一个开发者工具

*   **故事线**：OpenClaw 的作者并非传统程序员出身。他从第一行代码到产品上线，全程通过 AI Agent（如 Claude Code）以自然语言"指挥"完成。他做的事情本质上是：**写设计文档、定义需求、Review AI的输出、做取舍决策**——而不是敲代码。
*   **拆解关键环节**：
    *   如何用自然语言描述系统架构（而不是画UML图）
    *   如何让 Agent 自己跑测试、看报错、改代码的循环（Agentic Loop）
    *   当 Agent 写出"能跑但不优雅"的代码时，人的审美判断如何介入
    *   最终产物：一个被开发者社区广泛使用的终端Agent工具
*   **核心启发**："编程"正在被重新定义。会写代码当然好，但**不会写代码也不再是门槛**。门槛变成了：你能不能清晰地描述你要什么、判断交付物的质量。

### 案例2：Vibe Coding 实录——Live Demo 现场从零造一个产品

*   **出场工具**：`Claude Code` / `OpenClaw` / `MaxClaw` / `Kimi Claw`
*   **现场演示**：老师打开一个空目录，在终端里用纯自然语言，让 Agent 从零构建一个完整的小产品（例如：一个带计分板的网页贪吃蛇游戏 / 一个能实际使用的个人简历网站）。
*   **重点展示 Agent 的工作过程**（而非只看最终结果）：
    *   Agent 如何自己规划文件结构
    *   写到一半自己运行、发现报错、阅读错误日志
    *   自动生成修复补丁、重新测试
    *   最终成功弹出可用产品
*   **迭代演示**：产品做出来后，继续用自然语言追加需求——"加个暗黑模式""速度随分数递增""加背景音乐"——看 Agent 逐一消化并实现，展示"对话式迭代开发"的完整流程。
*   **核心启发**：未来的"编程"更像是当产品经理——你负责提需求、验收质量、决定方向，Agent负责执行。

### 案例3：Vibe Research——用AI重构你的学习和研究方式

*   **出场工具**：`NotebookLM` / `Kimi`
*   **现场演示**：
    *   将几篇晦涩的学术论文扔进 NotebookLM，一键生成**双人播客对谈音频**，现场播放30秒——枯燥的内容变成了轻松的闲聊。
    *   用 Kimi 的超长上下文能力，把一整本教材/一个完整的GitHub仓库扔进去，直接向它提问，让它成为这份材料的"活字典"。
*   **核心启发**：信息获取的方式正从"我去读"变成"它来讲给我听 / 我来直接问它"。

### 案例4：从想法到上线——3分钟用无代码平台搭建一个AI产品

*   **出场工具**：`扣子(Coze)` / `Dify`
*   **现场演示**：
    *   在 Coze 的可视化画布上，从零搭建一个实际可用的 AI Bot（比如一个"毒舌考研规划师"）。
    *   核心步骤展示：设定人设 Prompt → 挂载知识库（RAG） → 接入搜索插件 → 发布上线。
    *   生成二维码，大家可以现场扫码体验。
*   **核心启发**：**产品化**不再需要团队。一个人 + 一个下午 + 这些工具 = 一个可以服务真实用户的AI产品。

## 模块四：清醒认知——Agent 的能力边界与使用原则 (15分钟)

> 不灌鸡汤，讲清楚三件事。

### 1. Agent 会"翻车"

*   **幻觉问题**：Agent 会一本正经地编造不存在的API、引用不存在的论文。它不知道自己不知道。
*   **权限风险**：终端 Agent 拥有读写文件、执行命令的权限。一个模糊的"清理不需要的文件"指令，可能导致它删掉你的毕业论文。（可现场在沙盒环境中演示这个场景）
*   **死循环**：Agent 可能反复用同一个错误方法尝试修复同一个bug，陷入无限重试。

### 2. 安全底线

*   **沙盒隔离**：重要操作永远在容器/虚拟环境里运行，不要在宿主机上裸跑不受信任的Agent。
*   **Human-in-the-Loop**：涉及删除、支付、发布等不可逆操作，必须人工确认。像 Claude Code 默认就会在关键操作前暂停等你按 Y/N。
*   **不给敏感信息**：不要把密钥、密码直接写进 Prompt。

### 3. 真正值钱的能力

| 你以为值钱的 | 真正值钱的 |
|-------------|-----------|
| 会写代码 | 能把模糊需求拆解成清晰的、可执行的子任务 |
| 熟悉某个框架的API | 能判断AI生成的100个方案里哪个最优、为什么 |
| 打字快 | 能用精准的自然语言描述你到底要什么（Context Engineering） |
| 什么都自己干 | 知道什么该交给AI、什么必须自己把关 |

## 结尾 (5分钟)

*   **一个判断**：我们正处在这样一个时刻——"造东西"的门槛在急剧降低，但"造出好东西"的门槛从未降低。工具越强大，人的**品味（Taste）、判断力和问题定义能力**就越稀缺。
*   **一个建议**：今天回去，挑一个你生活中真实的低效场景（一个重复性的任务、一个你一直想做的小工具、一个你专业领域里的痛点），打开今天提到的任意一个工具，试着让AI帮你解决它。第一次可能不完美，但你会发现——**你已经有能力造东西了**。
*   **一句话**："AI won't replace you. But someone using AI will."
